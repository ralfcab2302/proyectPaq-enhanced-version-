2. Autenticación

La API utiliza JSON Web Tokens (JWT) para autenticar las peticiones. La mayoría de endpoints requieren un token válido en la cabecera HTTP Authorization.

2.1 Cabecera requerida

Authorization: Bearer <token>

2.2 Obtener un token — Login

POST /api/auth/login

Autentica al usuario y devuelve un token JWT.

Roles mínimos: Pública (sin autenticación)

Body (JSON)

Parámetro Tipo Requerido Descripción
correo string Sí Correo electrónico del usuario
contrasena string Sí Contraseña del usuario

Ejemplo de petición

curl -X POST https://backendcentral-desarrollo.up.railway.app/api/auth/login \

-H "Content-Type: application/json" \

-d '{"correo":"admin@paqtrack.com","contrasena":"admin123"}'

Respuesta exitosa (200)

{

"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

"usuario": {

"codigo_usuario": 1,

"correo": "admin@paqtrack.com",

"rol": "superadmin"

}

}

2.3 Ver perfil del usuario autenticado

GET /api/auth/perfil

Devuelve los datos del usuario autenticado.

Roles mínimos: usuario, admin, superadmin

Respuesta exitosa (200)

{

"codigo_usuario": 1,

"correo": "admin@paqtrack.com",

"rol": "superadmin",

"codigo_empresa": null

}

3. Empresas

Gestión del catálogo de empresas registradas en el sistema. Solo accesible por administradores y superadministradores.

3.1 Listar empresas

GET /api/empresas

Devuelve la lista de empresas. El superadmin ve todas; el admin solo la suya.

Roles mínimos: admin, superadmin

Respuesta exitosa (200)

[

{ "codigo": 1, "nombre": "GLS", "contacto": "admin@gls.com" },

{ "codigo": 2, "nombre": "SEUR", "contacto": "admin@seur.com" }

]

3.2 Detalle de una empresa

GET /api/empresas/:id

Devuelve los datos de una empresa concreta.

Roles mínimos: admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código de la empresa (codigo)

3.3 Crear empresa

POST /api/empresas

Crea una nueva empresa.

Roles mínimos: superadmin

Body (JSON)

Parámetro Tipo Requerido Descripción
nombre string Sí Nombre de la empresa
contacto string No Correo electrónico de contacto

Respuesta exitosa (201)

{ "codigo": 4, "nombre": "DHL", "contacto": "admin@dhl.com" }

3.4 Editar empresa

PUT /api/empresas/:id

Actualiza los datos de una empresa.

Roles mínimos: superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código de la empresa

Body (JSON)

Parámetro Tipo Requerido Descripción
nombre string No Nuevo nombre de la empresa
contacto string No Nuevo correo de contacto

3.5 Eliminar empresa

DELETE /api/empresas/:id

Elimina una empresa y todos sus datos asociados.

Roles mínimos: superadmin

⚠️ Esta operación elimina en cascada todos los usuarios y salidas de la empresa. Es irreversible.

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código de la empresa a eliminar

Respuesta exitosa (200)

{ "mensaje": "Empresa eliminada correctamente" }

4.1 Listar usuarios

GET /api/usuarios

Roles mínimos: admin, superadmin

Respuesta exitosa (200)

[

{ "codigo_usuario": 2, "correo": "admin@gls.com", "rol": "admin", "codigo_empresa": 1 },

{ "codigo_usuario": 3, "correo": "usuario1@gls.com", "rol": "usuario", "codigo_empresa": 1 }

]

4.2 Detalle de un usuario

GET /api/usuarios/:id

Devuelve los datos de un usuario.

Roles mínimos: admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código del usuario (codigo_usuario)

4.3 Crear usuario

POST /api/usuarios

Crea un nuevo usuario.

Roles mínimos: admin, superadmin

Body (JSON)

Parámetro Tipo Requerido Descripción

contrasena string Sí Contraseña (se almacena hasheada con bcrypt)

Respuesta exitosa (201)

{ "codigo_usuario": 10, "correo": "nuevo@gls.com", "rol": "usuario", "codigo_empresa": 1 }

4.4 Editar usuario

PUT /api/usuarios/:id

Roles mínimos: admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción

Body (JSON)

Parámetro Tipo Requerido Descripción
correo string No Nuevo correo electrónico
contrasena string No Nueva contraseña
rol string No Nuevo rol: admin o usuario

4.5 Eliminar usuario

DELETE /api/usuarios/:id

Elimina un usuario del sistema.

Roles mínimos: admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código del usuario a eliminar

Respuesta exitosa (200)

{ "mensaje": "Usuario eliminado correctamente" }

5. Salidas

Registro y consulta de los pases de paquetes por las salidas físicas de las máquinas clasificadoras. Todos los usuarios autenticados pueden consultar; solo los administradores pueden crear; solo el superadministrador puede eliminar.

5.1 Listar salidas

GET /api/salidas

Devuelve el listado de salidas con soporte de filtros y paginación.

Roles mínimos: usuario, admin, superadmin

Parámetros de query

Parámetro Tipo Requerido Descripción
nro_salida integer No Filtra por número de cinta (1-40)
desde string No Fecha de inicio en formato YYYY-MM-DD
hasta string No Fecha de fin en formato YYYY-MM-DD
pagina integer No Número de página (por defecto: 1)
limite integer No Resultados por página (por defecto: 20)

Ejemplo de petición

GET /api/salidas?nro_salida=5&desde=2026-01-01&pagina=1&limite=20

Respuesta exitosa (200)

{

"total": 45,

"pagina": 1,

"limite": 20,

"datos": [

{

"codigo": 101,

"codigo_empresa": 1,

"nro_salida": 5,

"codigo_barras": "ABC123456789",

"fecha_salida": "2026-01-15T10:32:00.000Z"

}

]

}

5.2 Estadísticas de salidas

GET /api/salidas/estadisticas

Devuelve un resumen estadístico de las salidas agrupado por cinta.

Roles mínimos: usuario, admin, superadmin

Respuesta exitosa (200)

[

{ "nro_salida": 1, "total_paquetes": 320 },

{ "nro_salida": 2, "total_paquetes": 285 }

]

5.3 Buscar por código de barras

GET /api/salidas/buscar/:codigoBarras

Devuelve el historial completo de un paquete a partir de su código de barras.

Roles mínimos: usuario, admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
codigoBarras string Sí Código de barras del paquete a consultar

Ejemplo de petición

GET /api/salidas/buscar/ABC123456789

Respuesta exitosa (200)

[

{

"codigo": 101,

"nro_salida": 5,

"fecha_salida": "2026-01-15T10:32:00.000Z",

"empresa": "GLS"

}

]

5.4 Detalle de una salida

GET /api/salidas/:id

Devuelve los datos completos de un registro de salida.

Roles mínimos: usuario, admin, superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código del registro de salida

5.5 Registrar una salida

POST /api/salidas

Registra el paso de un paquete por una salida de la máquina clasificadora.

Roles mínimos: admin, superadmin

Body (JSON)

Parámetro Tipo Requerido Descripción
nro_salida integer Sí Número de la salida física (1-40)
codigo_barras string Sí Código de barras del paquete
fecha_salida string No Fecha y hora en formato ISO 8601 (por defecto: now)

Ejemplo de petición

curl -X POST https://backendcentral-desarrollo.up.railway.app/api/salidas \

-H "Authorization: Bearer <token>" \

-H "Content-Type: application/json" \

-d '{"nro_salida":3,"codigo_barras":"XYZ987654321"}'

Respuesta exitosa (201)

{

"codigo": 302,

"codigo_empresa": 1,

"nro_salida": 3,

"codigo_barras": "XYZ987654321",

"fecha_salida": "2026-03-18T10:00:00.000Z"

}

5.6 Eliminar una salida

DELETE /api/salidas/:id

Elimina un registro de salida.

Roles mínimos: superadmin

Parámetros de ruta

Parámetro Tipo Requerido Descripción
id integer Sí Código del registro a eliminar

Respuesta exitosa (200)

{ "mensaje": "Salida eliminada correctamente" }

6. Códigos de Respuesta HTTP

Código Estado Descripción
200 OK Petición procesada con éxito
201 Created Recurso creado correctamente
400 Bad Request Parámetros inválidos o faltantes
401 Unauthorized Token ausente, inválido o expirado
403 Forbidden El usuario no tiene permisos para esta acción
404 Not Found El recurso solicitado no existe
500 Internal Server Error Error interno del servidor

Formato de error

Cuando la API devuelve un código de error, el cuerpo incluye siempre un campo mensaje:

{ "mensaje": "Descripción del error" }

7. Variables de Entorno

El servidor se configura mediante el archivo .env. A continuación se describen todas las variables disponibles:

Variable Valor por defecto Descripción
MYSQL_ROOT_PASSWORD root Contraseña root de MySQL
MYSQL_DATABASE paqtrack_db Nombre de la base de datos
JWT_SECRET clave_secreta_... Clave para firmar los tokens JWT
DB_HOST db Host de la base de datos (nombre del servicio Docker)
DB_USER root Usuario de MySQL
DB_PASSWORD root Contraseña del usuario de MySQL
DB_NAME paqtrack_db Base de datos a usar
PORT 3000 Puerto en el que escucha el servidor

⚠️ Cambia JWT_SECRET, DB_PASSWORD y MYSQL_ROOT_PASSWORD por valores seguros antes de desplegar en producción.
